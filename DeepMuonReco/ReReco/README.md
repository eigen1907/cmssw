# DeepMuonReco - ReReco Production

This directory contains the configurations and scripts required to perform the Re-Reconstruction (ReReco) step for the **DeepMuonReco** project. It includes tools to automate job submission to the Grid using CMS CRAB3.

## 📂 Directory Structure

Please ensure the directory is named **`ReReco`** to follow the naming convention.

```text
DeepMuonReco/
├── ReReco/
│   ├── config/
│   │   └── crab-dataset.json   # Input dataset list for CRAB (JSON format)
│   ├── scripts/
│   │   └── crab-submit.py      # Python script for automated CRAB submission
│   └── test/
│       ├── ReReco_cfg.py       # CMSSW configuration file (generated by cmsDriver)
│       ├── ReReco_driver.sh    # (Optional) Driver script for local testing
│       └── ReReco_run.sh       # (Optional) Shell execution script
└── ...
```

## ✅ Prerequisites
Before submitting jobs, ensure your environment is set up correctly:

### 1. CMSSW Environment: Ensure you are inside a valid CMSSW release.

```bash
cmsenv
```

### 2. CRAB3 Environment: Source the CRAB script.

```bash
source /cvmfs/cms.cern.ch/common/crab-setup.sh
```
### 3. VOMS Proxy: Initialize your grid proxy (valid for 1 week recommended).
```bash
voms-proxy-init --voms cms --valid 168:00
```

## 🚀 How to Submit CRAB Jobs
It is recommended to run the submission commands from inside the `ReReco` directory.

### 1. Configure Input Datasets
Edit `config/crab-dataset.json` to include the datasets you want to process.

- input_dataset: The full DAS name of the dataset.

- lumi_mask: Set to null for MC, or provide the URL/path for Data.

```json
[
  {
    "input_dataset": "/SingleMu_FlatPt-2to100/.../GEN-SIM-DIGI-RAW-MINIAOD",
    "lumi_mask": null
  }
]
```

### 2. Run the Submission Script
Use the `crab-submit.py` script to submit jobs for all datasets defined in the JSON file.
```bash
python3 scripts/crab-submit.py \
    -p test/ReReco_cfg.py \
    -i config/crab-dataset.json \
    -n DeepMuonReco_ReReco_v1 \
    -u joshin \
    -s T3_KR_UOS
```

#### Arguments Description
- `-p`, `--pset`: Path to the CMSSW configuration file (e.g., `test/RERECO_cfg.py`).

- `-i`, `--input`: Path to the JSON file containing input datasets.

- `-n`, `--name`: Project name. This will be part of the output directory structure in the storage element.

- `-s`, `--storage-site`: The CMS Storage Element (SE) where output files will be stored (e.g., `T2_KR_KISTI`, `T3_CH_CERNBOX`).

## 📊 Monitoring and Output

### Checking Job Status
After submission, a `crab_projects` directory will be created (usually where you ran the script). You can check the status using:

```bash
crab status -d crab_projects/crab_<Dataset_Name>_<Project_Name>
crab status -d ./crab_SingleMu_FlatPt-2to100_DeepMuonReco_ReReco_v1/
```

### Output Location
Successful jobs will store the output ROOT files at: `/store/user/<YourLXPLUSUsername>/<Project_Name>/<Dataset_Processed_Name>/...`

You can verify the files using `xrdfs` or `eos`:

```bash
xrdfs root://cms-xrdr.private.lo:2094/ ls /xrd/store/user/<username>/...
```
## 🛠 Troubleshooting
- **`ModuleNotFoundError`**: Ensure you have sourced the CRAB setup script (`source /cvmfs/...`).

- **Dataset not found**: Check if the dataset name in `crab-dataset.json` is correct and valid on DAS.

- **Permission Denied**: Ensure you have write permissions for the storage site specified in the `-s` flag.
